HPM Monitoring DATASET DESCRIPTION

1. Main description

There are 7 UNICEF section units or areas of activity, represented by different color tabs, with 7 main sheets. These sheets contain data on humanitarian activites by each UNICEF section. We call these sheets UNICEF sheets:

x
- CP (Child Protection),
- Education
- Nutrition
- WASH
- Health
- SBC (Social Behaviour and Change)
- SP (Social Protection)

For 5 of these 7 sections, they have an extra sheet called (Cluster). These sheets contains activities not only from UNICEF but from other NGO's that work on the same area of activity. We call these sheets Cluster sheets:

  • CP (Child Protection),

  • Education

  • Nutrition

  • WASH

There is also a sheet called Funding which will contain direct figures on funding status per month:

  • Section

  • Target population per section core indicator

  • Total funding requirements per section

  • Carry-over funding (from last year/exercise)

  • Funding received/available for each month

1.1 Internal table structure

The different tables in each of the sheets all have been defined as an Excel Table Object and named accordingly. Originally they are 1000 rows each, but being Excel tables, if the users add further rows at the end, the tables will automatically expand. The different tables have been named and their names and ranges can be seen in Excel from the 'Name manager':

The different table names correspond to the Tab names in lowercase and underscore.

2. Relevant fields of interest

  • All the UNICEF sheets have the same structure, as well as the Cluster sheets.

  • The difference comes only in the set of performance indicators that each Section uses. The set of indicators is small, ranging from 2 to 5 indicators. As a reference, There is an auxiliary hidden sheet indicators_lookup showing the indicator lookup tables for the dropdowns.

  • The list of provinces (11 in total) used in the dropdowns is listed in the auxiliary hidden sheet admin_lookup. These correspond to standard provinces names in Mozambique and PowerBi map engine should recognize and locate the identify the names easily. The map visualizations will not go below province level, so no need to worry about district names.

  • Some of the sheets contain some precalculated columns at the end, with yellow headers. These are only provided for the users to double check totals.

These will be the fields of interest when performing the initial dataload queries, as their values will be used across the different visualizations and data aggregations. 3. Data aggregation rules

3.1 Row-wise calculations (prior to column-wise aggregations)

Calculate subtotals and totals per row when not avaiable.

Some sections might provide very detailed data, but it is not the norm, As a suggestion is better to perform another operation to create table views as follows:

UNICEF Sheets:

For each row, you can follow this order:

  1. If total_boys figure is not provided, then sum boys_0-5_reached, boys_6-12_reached, boys_13-18_reached. If it is provided, then leave it as it is.

  2. If total_girls figure is not provided, then sum girls_0-5_reached, girls_6-12_reached, girls_13-18_reached. If it is provided, then leave it as it is.

  3. If total_children figure is not provided, then sum total_boys and total_girls. If it is provided, then leave it as it is.

  4. If total_pwd figure is not provided, the sum total_pwd_men and total_pwd_women. If it is provided, then leave it as it is.

  5. If total_adults figure is not provided, the sum total_men and total_women. If it is provided, then leave it as it is.

  6. If total_beneficiaries_reached is not provided, the sum total_children and total_adults. If it is provided, then leave it as it is.

CLUSTER Sheets:

Very similar but a little easier. For each row, you can follow this order:

  1. total_children as sumof total_boys and total_girls (if provided)

  2. If total_pwd as the sum total_pwd_men and total_pwd_women. (if provided)

  3. If total_adults as the sum total_men and total_women (if provided)

  4. If total_beneficiaries_reached is not provided, the sum total_children and total_adults. If it is provided, then leave it as it is.

3.2 Column-wise (grouping and aggregating)

For the dashboard visualizations, the row-wise population data previously calculated is expected to be aggregated for each separate section by month, province and indicator. In principle, the aggregaton operation is SUM.

IMPORTANT: once the population data is sum-aggregated (for each section) by month, province and indicator; it might eventually used in cumulative form in some of the visualizations. That is, if the month slicer selects for example 'January', 'February' and 'March', the result should be the cummulative across those months:

  • Example: 'Health' sector, for the indicator '# children 6 months to 15 years in humanitarian situations who are vaccinated against measles' in the province of 'Nampula' has total values of 10,000 beneficiaries in January, 20,000 in February and 15,000 in March; then when selecting 'March' in cumulative, the result would be 45,000

EXCEPTIONS: However there are a few specific indicators under sections SP and SBC in which the cummulative cannot be taken and the MAXIMUM across the selected months is taken: in this case for example, of the province of 'Nampula' has total values of 10,000 beneficiaries in January, 20,000 in February and 15,000 in March; then when selecting March total, the value would be 20,000 (maximum across these 3 months). I will provide you the names of the few indicators where this needs to be taken into account, as I have to confirm with the affected Sections (for the moment let's apply cummulative SUM across months in general

A suggestion could be to start grouping each UNICEF and Cluster datasheet following this grouping hierarchy:

  1. By month

  2. By province

  3. By indicator

Then the values of the numerical columns calculated previously in the rowwise operatons (boys, girls, etc) could be summed columnwise. This includes at least the total_beneficiaries column, but just in case, also other subtotals if available.

3.3 Additional columns after aggregation

Once total_beneficiaries data (plus other subtotals as specified in 3.1) has been aggregated by section, month, province and indicator for both UNICEF and Cluster sheets, I advice to add the following conditional-calculated columns, as they will be useful :

  • Month (by name): the reporting month contains the Excel numerical value (for example, 45323). It might be need to create a new column that reformats this date to a 'month name'

  • Quarter: (Q1, Q2, Q3 or Q4, depending on the value of month)

  • North_region: 'yes' if province belongs to [Cabo Delgado, Nampula or Niassa]; "no" for the rest of provinces

  • core_indicator: 'yes' if the indicators is a 'core' indicator; 'no' in case is not (the table is provided below)
